## Real-World Usage Scenarios

### Scenario 1: Open Source Project Monitoring
Monitor your OSS project alongside your Kubernetes infrastructure:
- Track community contributions
- See which PRs need attention (stale indicators)
- Watch for new issues that need triage

### Scenario 2: Development Team Workflow
Teams using GitHub for both code and Kubernetes:
- See release status while monitoring deployments
- Track PR velocity alongside deployment health
- Identify bottlenecks in the development pipeline

### Scenario 3: Multi-Cluster + Multi-Repo
Monitor repositories that deploy to your clusters:
- Infrastructure repo (Terraform, Helm charts)
- Application repos (microservices)
- Documentation and tooling repos
